## Executive summary

If you want one default path that works for most **human or mouse bulk RNA-seq** projects, make it this:

1. Plan the biology first: replication, RNA quality, library type, strandedness, and annotation version.
2. Run a reproducible workflow with **[nf-core/rnaseq](https://nf-co.re/rnaseq)** on **[Nextflow](https://www.nextflow.io/docs/latest/)**.
3. Treat **FastQC + MultiQC** as the universal first stop after FASTQ generation.
4. For expression-first work, quantify with **[Salmon](https://salmon.readthedocs.io/en/latest/)** and import with **[tximport](https://bioconductor.org/packages/tximport)**, ideally keeping provenance with **[tximeta](https://bioconductor.org/packages/tximeta)**.
5. Use **[DESeq2](https://bioconductor.org/packages/DESeq2)** as the default differential-expression engine.
6. Extract biology in layers: enrichment, pathway activity, TF activity, cell-state or cell-composition context, then splicing, isoforms, fusions, ASE, and regional analyses only when the experiment supports them.

This page is for people who already ran or are about to run a **bulk** RNA-seq experiment and want practical answers, not a method encyclopedia. Alternatives are included, but not treated as equal. The mainline recommendation is:

**Clean mammalian tissue, expression-first question, reasonable RNA quality**

`poly(A) enrichment -> stranded library -> FASTQ QC -> Salmon or nf-core/rnaseq -> tximport/tximeta -> DESeq2 -> fgsea/clusterProfiler + PROGENy + decoupleR/DoRothEA + cell-context inference`

Switch away from that default when the question changes:

- Choose **rRNA depletion** when RNA is degraded, FFPE-like, intronic signal matters, or non-polyadenylated transcripts matter.
- Choose **STAR** when you need genome alignments for splicing, fusions, allele-specific expression, or genome-browser coverage.
- Choose **paired-end and longer reads** as soon as exon usage, isoforms, splice events, or fusion detection become serious goals.

<Callout title="Who this is for" tone="success">
  <p>This guide stays strictly in bulk RNA-seq. If your core question depends on rare cell populations, lineage mixtures, or spatial localization, bulk RNA-seq may still be useful, but a single-cell, spatial, or long-read assay may answer the real question better.</p>
</Callout>

<Callout title="Poly(A) versus rRNA depletion" tone="warning">
  <p><strong>Poly(A) enrichment</strong> is the cleanest default for coding-focused mammalian expression studies with decent RNA integrity.</p>
  <p><strong>rRNA depletion</strong> is the better default for degraded RNA, FFPE-like material, non-polyadenylated transcripts, nascent or intronic signal, and broader transcriptome coverage. It often trades interpretive simplicity for coverage breadth.</p>
</Callout>

<Callout title="Human and mouse annotation note">
  <p>For computation, prefer stable IDs such as Ensembl gene identifiers. Use gene symbols for figures and readable tables later. Strip version suffixes only intentionally, because careless version stripping can silently merge or drop rows.</p>
  <p>If you map mouse genes into human-centric signatures, treat ortholog mapping as a convenience layer. It is not ground truth, especially for immune, developmental, and rapidly evolving gene families.</p>
</Callout>

### Annotation and reference strategy

Use one reference bundle and freeze it:

- **[GENCODE](https://www.gencodegenes.org/)**: strong default for human and mouse when you want well-curated gene and transcript models.
- **[Ensembl](https://www.ensembl.org/info/genome/genebuild/index.html)**: strong default when your ecosystem or annotation tooling is already Ensembl-centric.
- **[NCBI RefSeq](https://www.ncbi.nlm.nih.gov/refseq/)**: use when a collaborator, institution, or downstream clinical context is already standardized on RefSeq.

Do not mix:

- GENCODE transcriptome index with RefSeq gene annotation in downstream joins
- Ensembl IDs from one release with symbols from another release
- human-centric signature resources onto mouse data without explicitly documenting ortholog conversion

### Replication, power, and controls

Biological replication matters more than squeezing one more downstream algorithm out of underpowered data.

- Use **[RNASeqPower](https://bioconductor.org/packages/RNASeqPower)** or **[powsimR](https://bvieth.github.io/powsimR/)** when you can still influence the experiment.
- If you are already underpowered, be honest about effect-size uncertainty and treat pathway-level summaries as supportive, not rescuing.
- ERCC-style spike-ins can be useful in selected designs, but they do not replace good biological replication and they can create a false sense of security if used as a generic fix.

```r
# Simple planning logic, not a substitute for study-specific assumptions
library(RNASeqPower)
est_power <- rnapower(
  depth = 20,
  n = 4,
  cv = 0.3,
  effect = 1.5,
  alpha = 0.05
)
est_power
```

<Callout title="Planning pitfall">
  <p>Do not treat batch as an afterthought. If all treated samples were processed on a different day, by a different person, or with a different kit lot, that is a design problem first and a statistics problem second.</p>
</Callout>

## From FASTQ to trusted quantification

Keep the data type explicit at each step:

- `FASTQ`: raw reads and base qualities
- `BAM/CRAM`: genome alignments
- `quant.sf` or equivalent: transcript-level abundance estimates
- gene count matrix: integer counts for count-based DE frameworks
- normalized matrix: transformed expression for visualization and sample structure
- ranked vector: gene-wise statistics for GSEA
- splice-event or exon tables: event-aware follow-up
- fusion, ASE, regional scores: specialized outputs

- **[RSeQC](http://rseqc.sourceforge.net/)**: `infer_experiment.py`, `read_distribution.py`, `geneBody_coverage.py`

### UMI branch for bulk libraries

UMIs are uncommon in standard bulk RNA-seq, but they do exist in targeted or specialized bulk designs. If UMIs are present, handle them explicitly:

- Use **[UMI-tools](https://umi-tools.readthedocs.io/en/latest/)** for extraction and deduplication.
- If you run **nf-core/rnaseq**, keep the UMI-aware outputs and deduplication settings visible in your provenance.
- Deduplication is not an optional cosmetic step if the library was designed around UMIs.

```bash
umi_tools extract \
  --bc-pattern=NNNNNNNN \
  -I sample_R1.fastq.gz \
  --stdout extracted_R1.fastq.gz \
  --read2-in sample_R2.fastq.gz \
  --read2-out extracted_R2.fastq.gz
```

## Core analysis

Once you have a count-like object and trustworthy metadata, the work becomes statistical and interpretive rather than purely computational.

### Import, normalization, and sample structure

For most users, the best default is **DESeq2** on imported gene-level counts. If you quantified with Salmon, do not throw away transcript-level work by flattening it carelessly. Use `tximport`, and use `tximeta` when you can.

```r
library(DESeq2)
library(tximeta)

se <- tximeta(read.csv("metadata/samples.csv"))
gse <- summarizeToGene(se)

dds <- DESeqDataSet(gse, design = ~ batch + condition)
dds <- DESeq(dds)

vsd <- vst(dds, blind = FALSE)
plotPCA(vsd, intgroup = c("condition", "batch"))
```

<Callout title="Count models versus voom">
  <p><strong>DESeq2</strong> and <strong>edgeR</strong> model counts directly. <strong>voom</strong> estimates mean-variance weights from count data and passes those weights into limma's linear-model framework. None of these frameworks will rescue a confused design formula.</p>
</Callout>

## Common pitfalls

This section is intentionally blunt. Most bulk RNA-seq pain comes from a small set of repeated mistakes.

### Technical and analytical pitfalls

1. **No real replication.** Statistical sophistication does not replace biological replication.
2. **Annotation drift.** If the index, tx2gene mapping, and visualization symbols came from different annotation releases, joins will break or silently lie.
3. **Wrong strandedness assumption.** Use `infer_experiment.py` or equivalent checks. Counting with the wrong strand setting can quietly invert confidence in some loci.
4. **Blind trimming.** Trim because there is a problem, not because it feels hygienic.
5. **Ignoring multi-mapping.** Repetitive genes, paralogs, pseudogenes, and short-read ambiguity are biological and computational realities.
6. **Over-correcting batch.** Batch correction can remove signal of interest if design and biology are entangled.
7. **Short reads for structure questions.** If the goal is transcript structure, exon usage, or fusion discovery, cheap short single-end data are often the real limitation.
8. **rRNA contamination denial.** High rRNA fractions are not just annoying; they waste depth and can indicate library problems.
9. **Assuming weak signal means no biology.** It can also mean wrong tissue compartment, weak perturbation, bad timing, or overwhelming composition changes.
10. **Treating every advanced method as mandatory.** Most projects do not need fusions, ASE, PREDA, and WGCNA all at once.

### What to do when signal is weak or libraries look bad

- Re-check sample labels and metadata first.
- Re-check strandedness and annotation consistency.
- Inspect gene-body coverage and read distribution.
- Ask whether the perturbation timing or tissue compartment was wrong.
- Look at known positive-control genes or pathways before declaring failure.
- Consider that composition shifts may dominate weak within-cell-state changes.
- If everything depends on rare populations or spatial structure, bulk may not be the right assay.

### Interpretation pitfalls

<Callout title="Interpretation principle" tone="warning">
  <p>Expression changes reflect what cells are responding to or attempting, not necessarily what is functionally occurring.</p>
</Callout>

1. **Compensatory versus functional change.** Upregulation does not imply more function. It may reflect compensation after stress, damage, or failed homeostasis.
2. **Cell composition versus cell state.** Differences can come from more cells of one type, the same cells in a different state, or both.
3. **Structure versus expression mismatch.** Higher mRNA does not guarantee correct protein structure, localization, or tissue architecture.
4. **Stress and injury responses are nonspecific.** Hypoxia, inflammation, interferon, EMT-like programs, and immediate-early responses are reused across many contexts.
5. **Pathway redundancy.** Overlapping gene sets can make several pathways look independently significant when they really reflect one underlying program.
6. **Directionality ambiguity.** Up or down does not automatically define activation, inhibition, or biological outcome.
7. **Bulk averaging.** Signals are averages across mixed populations. Rare or spatially restricted changes can disappear.
8. **mRNA is not protein and not function.** Translation, post-translational regulation, localization, and turnover matter.
9. **Temporal ambiguity.** A single time point cannot cleanly separate cause from consequence.
10. **Reverse causality.** Especially in tumor and inflammatory settings, observed changes may be downstream responses rather than drivers.
11. **Signatures are not mechanisms.** Enrichment indicates similarity to a known program, not mechanistic activation.
12. **Cross-species limitations.** Mouse-to-human ortholog mapping loses nuance and sometimes meaning.

## Final recommended path

If you want the shortest honest recommendation:

1. Design the experiment around the biological question, not around whichever downstream package you hope to use.
2. For most human and mouse bulk RNA-seq projects, use **stranded poly(A)** if RNA quality is good and the question is coding-gene expression.
3. Start every dataset with **FastQC + MultiQC**, then add **RSeQC**, **RNA-SeQC**, or **Picard** when RNA-specific sanity checks matter.
4. Use **Salmon + tximport + tximeta** for expression-first work.
5. Use **STAR** when you genuinely need genome-aware analyses.
6. Use **DESeq2** for gene-level DE, then add **lfc shrinkage**, **fgsea**, **PROGENy**, **decoupleR/DoRothEA**, and a conservative cell-context layer.
7. Treat splicing, isoforms, fusions, ASE, and regional analyses as targeted follow-up, not default decorations.
8. Freeze annotation versions, freeze metadata, and save exportable artifacts before you move on.
